## 개요

Google Search Console은 사이트의 검색 성능을 모니터링하고 최적화하는 필수 도구입니다.
정기적인 모니터링을 통해 SEO 이슈를 조기에 발견하고 대응할 수 있습니다.

### 주요 모니터링 영역

1. **커버리지 보고서**: 색인 생성 상태 확인
2. **실적 보고서**: 검색 트래픽 분석
3. **URL 검사**: 개별 페이지 문제 진단
4. **Core Web Vitals**: 페이지 경험 지표

## 1. 커버리지 보고서 정기 점검

### 점검 주기

- **매주 월요일 오전 10시**
- 캘린더 알림 설정 권장

### 점검 프로세스

#### 1.1 색인 생성 상태 확인

1. Google Search Console 접속: <https://search.google.com/search-console>
2. 속성 선택: `blog.jell.kr`
3. 좌측 메뉴 > **색인 생성** > **페이지** 클릭

#### 1.2 확인 항목

**✅ 정상 상태:**

- "색인이 생성됨" 페이지 수가 예상 페이지 수와 일치
- 새로 발행한 글이 24시간 내 색인 생성됨
- 오류 및 경고 수가 0개 또는 안정적

**⚠️ 주의 필요:**

- 색인 생성 페이지 수가 갑자기 감소
- "색인이 생성되지 않음" 페이지 수 증가
- "크롤링됨 - 현재 색인이 생성되지 않음" 경고 발생

**🚨 긴급 대응 필요:**

- "오류" 항목에 페이지가 나타남
- robots.txt 차단 오류
- 404 오류 급증
- 서버 오류(5xx) 발생

#### 1.3 이슈 발견 시 대응

**Step 1: 문제 유형 파악**

```text
오류 유형별 대응:

1. robots.txt 차단
   → robots.txt 파일 확인
   → 의도하지 않은 차단 규칙 제거

2. noindex 태그
   → 해당 페이지의 meta 태그 확인
   → 불필요한 noindex 제거

3. 404 오류
   → 링크 끊어진 페이지 확인
   → 리다이렉트 또는 페이지 복구

4. 서버 오류(5xx)
   → 서버 로그 확인
   → 호스팅 상태 점검 (Netlify 대시보드)

5. 크롤링 차단
   → CDN 설정 확인
   → 방화벽 규칙 검토
```

**Step 2: 이슈 기록**

문제 발생 시 다음 정보를 기록:

```markdown
## [날짜] GSC 이슈 로그

### 문제 상황

- 발견 일시:
- 문제 유형:
- 영향받은 페이지 수:
- 페이지 예시:

### 원인 분석

[원인 설명]

### 해결 방법

[적용한 해결책]

### 결과

[해결 후 상태]
```

#### 1.4 주간 체크리스트

```markdown

- [ ] 색인 생성 페이지 수 확인: \_\_\_개

- [ ] 오류 페이지: \_\_\_개
- [ ] 경고 페이지: \_\_\_개
- [ ] 제외된 페이지: \_\_\_개
- [ ] ## 특이사항 및 조치사항:
```

## 2. URL 검사 프로세스

### 2.1 URL 검사가 필요한 상황

1. **새 글 발행 후**
   - 색인 생성 요청을 통해 빠른 노출

2. **콘텐츠 수정 후**
   - 업데이트된 내용을 Google에 알림

3. **문제 페이지 발견 시**
   - 구체적인 오류 원인 파악

4. **sitemap 제출 후**
   - 주요 페이지의 색인 상태 확인

### 2.2 URL 검사 실행 방법

1. Google Search Console 상단 검색창에 URL 입력
2. 전체 URL 입력 (예: `https://blog.jell.kr/posts/dev/js/...`)
3. **Enter** 키 입력

### 2.3 검사 결과 해석

#### ✅ "URL이 Google에 등록되어 있음"

```text
상태: 정상
조치: 없음

확인 사항:
- 마지막 크롤링 날짜
- 사이트맵에서 제출 여부
- 모바일 사용 편의성
- Core Web Vitals 점수
```

#### ⚠️ "URL이 Google에 등록되어 있지 않음"

```text
상태: 색인 대기 중 또는 문제 있음

확인 사항:
1. "색인 생성 요청" 버튼 클릭 가능 여부
2. 차단 사유 확인:
   - robots.txt 차단
   - noindex 태그
   - 정규 URL 문제
   - 크롤링 오류

조치:
- 문제가 없다면: "색인 생성 요청" 클릭
- 문제가 있다면: 해당 이슈 해결 후 재요청
```

#### 🚨 "URL에 문제가 있음"

```text
상태: 긴급 조치 필요

일반적인 오류:
1. 서버 오류 (5xx)
   → 호스팅 상태 확인

2. 리디렉션 오류
   → 리디렉션 체인 점검

3. robots.txt 차단
   → robots.txt 설정 수정

4. noindex 태그
   → 메타 태그 제거

5. 소프트 404
   → 실제 404 상태 코드 반환 또는 콘텐츠 추가
```

### 2.4 색인 생성 요청 프로세스

**Step 1: 사전 확인**

```bash
# 로컬에서 페이지 접근 가능 여부 확인
curl -I https://blog.jell.kr/posts/[slug]

# 예상 응답: HTTP/2 200
```

**Step 2: 색인 생성 요청**

1. URL 검사 결과 페이지에서 **"색인 생성 요청"** 버튼 클릭
2. 1-2분 대기 (실시간 테스트 진행)
3. 결과 확인

**Step 3: 결과 기록**

```markdown
## 색인 생성 요청 로그

- 요청 일시: YYYY-MM-DD HH:MM
- URL: [페이지 URL]
- 요청 사유: [신규 발행 / 콘텐츠 수정 / 오류 수정]
- 결과: [성공 / 실패]
- 비고:
```

### 2.5 색인 생성 템플릿

**새 글 발행 시 체크리스트:**

```markdown
## 신규 글 색인 체크리스트

게시물: [제목]
URL: [전체 URL]
발행 일시: YYYY-MM-DD HH:MM

- [ ] 24시간 후 색인 상태 재확인
- [ ] 실적 보고서에서 노출 시작 확인 (3-7일 후)
```

**콘텐츠 수정 시 체크리스트:**

```markdown

게시물: [제목]

수정 일시: YYYY-MM-DD HH:MM

- [ ] 변경사항 배포 확인
- [ ] GSC에서 URL 검사 실행
- [ ] 색인 생성 요청 제출
- [ ] "실시간 테스트" 통과 확인
- [ ] 48시간 후 색인 업데이트 확인
```

#### 총 클릭수

```text
의미: 검색 결과에서 사이트로 유입된 횟수
목표: 전주/전월 대비 5% 이상 성장
```

```text
의미: 검색 결과에 사이트가 표시된 횟수
목표: 지속적 증가 추세 유지
경고: 갑작스러운 감소 시 색인 문제 의심
```

#### 평균 CTR (클릭률)

```text
의미: 노출 대비 클릭 비율 (클릭수 / 노출수 × 100)
기준:
  - 우수: 5% 이상
  - 보통: 2-5%
  - 개선 필요: 2% 미만

개선 방법:
- 메타 설명 최적화
- 제목 태그 개선
- 구조화된 데이터 추가
```

#### 평균 게재순위

```text
의미: 검색 결과에서의 평균 위치
목표: 10위 이내 (첫 페이지)
우수: 3위 이내

개선 방법:
- 콘텐츠 품질 향상
- 키워드 최적화
- 내부 링크 강화
- 백링크 확보
```

## 4. 모니터링 자동화

### 4.1 알림 설정

Google Search Console은 이메일 알림을 제공합니다:

1. GSC 설정 > **사용자 및 권한** > **이메일 알림 설정**
2. 다음 항목 활성화:
   - ✅ 중요한 문제 (보안 문제, 수동 조치)
   - ✅ 모바일 사용 편의성 문제
   - ✅ 새 메시지
   - ✅ 색인 생성 문제

### 4.2 정기 점검 일정

```text
일일:
- 이메일 알림 확인 (자동)

주간 (매주 월요일 오전 10시):
- [ ] 커버리지 보고서 확인
- [ ] 실적 보고서 주간 분석
- [ ] 주간 보고서 작성

월간 (매월 1일):
- [ ] 실적 보고서 월간 분석
- [ ] 트렌드 분석 및 인사이트 도출
- [ ] 다음 달 목표 설정
```
